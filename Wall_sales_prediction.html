Introduction¶

Abstract: The following is a one afternoon work-out to use regression modeling and Monte Carlo (more precisely Metro Polis algorithm) to construct a predicting model. We then use this tool to forcast the sales value of a retailer, the "Wall", 8 months in future.¶

Data structure¶

the data are time series of historical sales for different department at different stores although accompanied to these set are other informations that can be considered in correlation to the sales of a given department, we only filter out one department in one single store and work on a model based only on the historical sales of this department.

Objective and the data¶

The given sales are from $\mathbf{2010-02-05}$ until $\mathbf{2012-10-26}$ and reported weekly.

Objective¶

So, we want to split the data in two sets one running from $\mathbf{2010-02-05}$ up to $\mathbf{100}$ weeks and one running from week $\mathbf{100}$ to the end which is $\mathbf{2012-10-26}$. The latter had $\mathbf{39}$ weeks. To be generic, I use $\mathbf{T}$ for the length of the period we are interested $\mathbf{t}$ forecast.

The first set is used to train a model and the second set is assumed to be the test set where we predict its sales using the trained model.

Methodology¶

There are probably many ways to construct a model. Here we are going to be simple and yet creative. Assume a the period ${\bf t_0 }$ to ${\bf t_0+T }$, where ${\bf T }$ is the length of the period we are forecasting. To put this in number, we may pick $${\bf t_0=\textit{2011-02-05} }$$ and thus $${ \bf t_0 + T = \textit{2010-02-05} + 39\textit{ weeks } }$$.

Also I am going to pick a period ${\bf t_0 - T}$ up to ${\bf t_0 }$ and assume that the sales from ${\bf t_0 - T }$ up to ${ \bf t_0 }$ (regressors) predict the sales in ${\bf t_0}$ up to ${\bf t_0+T}$ (targets).

In the first ${\bf 100 }$ weeks (our training set) we could construct $\mathbf{num_{train} = 22}$ rows of observations that is: \begin{eqnarray} \\ \mathbf{X} = \begin{bmatrix} 1 & 1 & \dots \\ x_0^{t_0} & x_0^{t_0 + 1} & \dots \\ \vdots & \vdots & \dots \\ x_{-T}^{t_0} & x_{-T}^{t_0 + 1} & \dots \end{bmatrix} \end{eqnarray}

\begin{eqnarray} \\ \mathbf{Y} = \begin{bmatrix} y_T^{t_0} & y_T^{t_0 + 1} & \dots \\ \vdots & \vdots & \dots \\ y_{0}^{t_0} & y_{0}^{t_0 + 1} & \dots \end{bmatrix} \end{eqnarray}

The model is defined as a set of parameters $\theta$ such that: \begin{eqnarray} \\ \mathbf{\theta}_{ij} \mathbf{X}_{jm} = \mathbf{Y}_{im} + \mathbf{\epsilon}_{im} \end{eqnarray}

Obviously we do not no first hand what the ranges of these parameters (${\bf \theta_{ij} }$) are. Also the number of them are pretty much high. So, I Assume that the parameters (${\bf \theta_{ij} }$) can be considered discrete values and I specify a default upper and lower bond. These parameters, physically take the form af a classical spins. Each spin takes an $N$ discrete values. Finally a Monte Carlo simulation is used to find the ground state of the spins, assuming a energy (the error function). The energy (or the error): \begin{eqnarray} \\ \mathbf{H} = \sqrt{\mathbf{\epsilon} ^{T} \,\, . \,\, \mathbf{\epsilon}} \end{eqnarray}

'''
 Here are the function used todo the optimization:
 1) getError(theta, X, Y) is basically the hamiltonian of the model. It takes observations X and Y in and compute 
     the hamiltoninan (error) given a set of theta parameters or physically a set of spin configurations "theta".
'''
def getError(theta, X, Y):
    E = (theta.dot(X) - Y)
    E = E.ravel()
    return np.sqrt(E.dot(E.transpose())[0,0])
    
'''
2) although this is not used, it actually initializzes the theta randomly given a lower (min_theta) and 
    uper bound (max_theta) for each spin (a theta entry) and an increment (dTheta) 
    that specifies the spacing between "min_theta" and "max_theta".
'''    

def initializeTheta(theta, 
                    dTheta = 0.25,
                    min_theta=-1, 
                    max_theta=+1, 
                    ):
    for i in range(theta.shape[0]):
        for j in range(theta.shape[1]):
            r = np.random.uniform(min_theta, max_theta+dTheta)
            val = min_theta + dTheta * int((r-min_theta)/dTheta)
            theta[i,j]=val
    return theta

'''
3) the update function takes a current theta and randomly seek to change one entry 
    to a random value within the bounds. If the new theta results in a lower energy the 
    new theta replaces the current theta.
'''
            
def update(theta,
           X,
           Y,
           dTheta = 0.25,
           min_theta=-1, 
           max_theta=+1):
    error_current = getError(theta, X, Y)
#    print error_current
    
    i = np.random.randint(theta.shape[0])
    j = np.random.randint(theta.shape[1])
    n = np.random.randint(int((max_theta - min_theta)/dTheta)+1)
    theta_new = n*dTheta + min_theta
    
    theta_current  = theta[i,j] 
    theta[i,j]  = theta_new
    
    error_new = getError(theta, X, Y)
#    print error_new
    if error_new< error_current:
        error_current = error_new
    else:
        theta[i,j]=theta_current        

    return error_current

'''
4) And finally the core function that is the Monte Carlo. It goes over a number of sampling steps 
    num_mc and updates the theta using the "update" function. 
'''    
    
def optimize(theta, 
             X, 
             Y, 
             num_mc = 1000,
             dTheta = 0.25,
             min_theta=-1, 
             max_theta=+1
             ):
    
    errors=[]
    for n_mc in range(num_mc):
        error = update(theta,
               X,
               Y,
               dTheta,
               min_theta, 
               max_theta,
               )
        errors.append(error)
    return errors
